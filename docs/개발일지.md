# 개발일지 — 서울 자율주행버스 실시간 위치

## 2026-06-15 — 초기 구성 (React + Supabase + 카카오 로그인)

### 한 일
- Claude Design 시안(HTML/CSS/JS)을 **Vite + React 18 + 순수 CSS**로 이식.
  - `styles.css` 전체를 `src/styles/index.css`로 옮기고, 인증/즐겨찾기 UI 스타일 추가.
  - 정적 마크업을 컴포넌트로 분리(Header, LineNav, RoadLayer, Hero, LiveMap, About,
    RoutesSection, HowTo, Stats, Faq, Footer).
  - 모든 카피를 `data/content.js`에 `[ko, en]` 형태로 단일화하고, `LanguageContext`의
    `t(ko, en)`로 렌더. (원본의 `data-en` innerHTML 스왑 방식 → React 상태 기반 i18n으로 교체)
- 임퍼러티브 엔진은 훅으로 이식(마크업은 React가, 좌표/애니메이션은 훅이 구동):
  - `useRoadEngine` — `road-engine.js` 이식. 문서 전체 S-커브 도로 생성 + 스크롤에 따라
    버스가 정류장에 직립 정차. 언어 변경 시 레이아웃 높이 변동 → `__roadRebuild` 재호출.
  - `useLiveMap` — `app.js`의 LIVE DEMO MAP 이식. 데모 버스 3대 순환, 정류장/혼잡도/ETA
    실시간 갱신. 선택 차량을 `onSelect`로 React에 노출(즐겨찾기 연동용).
  - `useReveal` — 스크롤 진입 시 `.reveal.in`.
  - FAQ 토글·스탯 카운트업·시계는 React 상태로 재작성.
- **Supabase + 카카오 로그인**:
  - `lib/supabase.js` — 환경변수 있을 때만 클라이언트 생성(`isSupabaseConfigured`).
  - `lib/auth.js` — `signInWithOAuth({ provider: 'kakao' })`.
  - `hooks/useAuth.js` — `getSession` + `onAuthStateChange` 세션 추적.
  - `components/AuthControls.jsx` — 헤더의 카카오 로그인 버튼 / 로그인 후 프로필 칩·팝오버.
- **즐겨찾는 차량**:
  - `lib/favorites.js` — `favorite_buses` 테이블 CRUD(RLS). LiveMap에서 선택 차량 토글.

## 2026-06-16 — 채팅 도우미 (Solar + OpenAI / Edge Function)

### 한 일
- **GitHub Pages 404 수정**: Pages 소스가 `legacy`(브랜치 직접 배포)로 잡혀 원본
  `index.html`(→ `/src/main.jsx`)을 서빙하던 문제. `build_type`을 `workflow`로 전환 후
  재배포. 라이브에서 빌드 번들(`/rest06/assets/*`) 200 확인.
- **우하단 채팅 팝업** 추가:
  - `supabase/functions/chat/index.ts` — Edge Function 프록시. 요청 토큰으로
    `auth.getUser()` 검증(로그인 사용자만), **Solar 우선 → 실패 시 OpenAI 폴백**.
    키는 함수 secret(SOLAR_API_KEY/OPENAI_API_KEY)에서만 읽어 브라우저 비노출.
    모델은 `SOLAR_MODEL`/`OPENAI_MODEL` secret으로 교체 가능(기본 solar-pro2 / gpt-4o-mini).
  - `lib/chat.js` — `functions.invoke('chat', { body:{messages} })`. supabase-js가 토큰 자동 첨부.
  - `components/ChatWidget.jsx` — 런처 버튼 + 팝업(메시지 목록·입력·타이핑 인디케이터).
    비로그인 시 카카오 로그인 안내. KO/EN 대응.
  - 위젯 스타일 `styles/index.css`에 추가, `App`에 마운트.

### 결정·함정
- 정적 호스팅이라 키를 클라이언트에서 읽으면 노출 → **Edge Function 프록시**로 결정.
- Solar 공식 문서가 두 키를 함께 다루는 점을 반영해 함수에 두 키를 모두 보관하고
  Solar→OpenAI 폴백 구조로 구성. Solar는 OpenAI 호환 `/v1/chat/completions` 사용.
- 플랫폼 `verify_jwt`는 anon 키도 통과하므로, 실제 로그인 구분은 `getUser()`로 처리.

### 검증
- `npm run build` 성공(97 modules). 함수 배포·secret 설정은 사용자 환경에서 1회 필요(README).
