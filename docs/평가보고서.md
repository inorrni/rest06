# 평가보고서 — 서울 자율주행버스 실시간 위치 (React 이식)

## 점수표
| 항목 | 점수 | 비고 |
|---|---|---|
| 디자인 충실도 | 9 / 10 | 원본 CSS·모션·도로 엔진·라이브맵 그대로 이식 |
| 컴포넌트 구조 | 9 / 10 | 섹션별 분리 + 데이터 기반 렌더 + 단일 콘텐츠 소스 |
| i18n(KO/EN) | 9 / 10 | DOM 스왑 → Context 기반 `t()` 로 React 친화적 재작성 |
| Supabase 연동 | 8 / 10 | 클라이언트/세션/즐겨찾기 + 미설정 가드. 실데이터는 데모 |
| 카카오 로그인 | 8 / 10 | Supabase OAuth provider 방식. 콘솔 설정은 사용자 몫(문서화) |
| 빌드/안정성 | 9 / 10 | 빌드 통과, StrictMode 재실행 안전(cleanup 완비) |

## 강점
- 임퍼러티브 시각 엔진(스크롤 도로 + 라이브맵)을 **훅으로 캡슐화**해 React 마크업과 깔끔히 분리.
- 모든 문구를 `data/content.js` 한 곳에 모아 KO/EN·콘텐츠 수정 용이.
- Supabase 키가 없어도 사이트가 깨지지 않는 **graceful degradation**.
- 로그인 가치 제공: 차량 즐겨찾기를 RLS 테이블에 저장.

## 개선 여지
- OG 이미지/파비콘, GitHub Pages 배포 워크플로 미구성(다음 작업).
- 라이브맵은 데모 데이터 — 실제 차량 텔레메트리 API 연동 필요.
- `howto-note`의 🚌 는 원본 카피 유지를 위해 그대로 둠(아이콘 SVG 대체 검토 가능).
- 카카오 프로필 메타데이터 키 매핑은 provider 응답에 따라 조정 필요할 수 있음.

## 리스크
- 카카오 OAuth는 Supabase provider + Kakao Developers Redirect URI 설정이 선행되어야 동작.
- 도로 엔진은 문서 높이/폰트 로드에 의존 → 지연 rebuild로 보정하나 극단적 레이아웃에서 재점검 권장.

## 결론
시안을 기능 손실 없이 React로 이식했고, Supabase 카카오 로그인과 즐겨찾기까지 동작 가능한
형태로 연동했다. 키·테이블·OAuth 콘솔 설정만 채우면 즉시 로그인 플로우가 살아난다.
